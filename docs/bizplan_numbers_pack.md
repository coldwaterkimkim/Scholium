### 1-2. 예비창업자 여부 / 매출 / 이익 / 자본금 기입 방식

이 항목은 repo/artifact만으로는 확정할 수 없다. 아래처럼 기입하는 게 안전하다.

| 항목 | 기입 방식 |
| --- | --- |
| 예비창업자 여부 | `확인 필요` |
| 매출 | 현재 결제/정산/매출 증빙 근거가 repo에 없으므로, 실제 매출이 없으면 `0원` 기입 초안 |
| 영업이익 또는 당기순이익 | 실제 매출이 없고 비용만 발생했다면 음수 또는 `0원` 여부를 실제 지출 기준으로 입력. 숫자 미정이면 `확인 필요` |
| 자본금 | 법인 미설립이면 `0원` 또는 양식상 `해당없음`, 법인 설립 상태면 실제 납입자본금 입력 |

### 1-3. 가장 보수적인 현재 상태 기입 초안

아래는 실제 사업자/법인 등록과 매출이 아직 없다는 가정일 때만 쓰는 초안이다.

| 항목 | 보수 초안 | 표시 방식 |
| --- | --- | --- |
| 예비창업자 여부 | `예` | `가정` |
| 2025 매출 | `0원` | `가정` |
| 2025 영업이익 | `0원` 또는 `확인 필요` | 실제 개인 지출 처리 방식에 따라 수정 필요 |
| 자본금 | `0원` | 법인 미설립 가정일 때만 사용 |

## 3. 조달 계획 초안

단위: `백만원`

원칙:
- 매출은 아직 확정 근거가 약하므로 초반 조달에서 비매출 비중을 높게 잡는 게 안전하다.
- 2026은 정부지원금/자기부담, 2027 이후는 파일럿 매출과 후속 조달을 섞는 방식이 보수적이다.

| 항목 | 2026 | 2027 | 2028 | 메모 |
| --- | ---: | ---: | ---: | --- |
| 정부지원금/사업화 자금 | 70 | 60 | 40 | `가정` |
| 창업자 자기부담/초기 투입 | 30 | 20 | 10 | `가정` |
| 엔젤/브릿지 투자 | 0 | 80 | 120 | `가정` |
| 파일럿/초기 매출 | 0 | 40 | 130 | 과대추정 피하려고 보수적으로 둠 |
| 합계 | 100 | 200 | 300 | 위 자금 소요 초안과 맞춤 |

조달 해석:
- 2026:
  - 제품 고도화 + benchmark/QA + 초기 고객검증 단계
  - 매출보다는 지원금/자기부담 중심이 안전
- 2027:
  - 기관 파일럿과 후속 조달 병행
- 2028:
  - 파일럿 매출과 초기 반복매출 비중 확대 가정

## 4. 고용 계획 초안

주의:
- 현재 repo/artifact로는 실제 팀 인원 확정 근거가 없다.
- 아래는 "지금 구조에서 3년 안에 가장 무리 없는 채용 순서" 기준의 `가정 초안`이다.

| 연도 | 연말 기준 인원 초안 | 역할 | 메모 |
| --- | ---: | --- | --- |
| 현재 | `확인 필요` | 대표/개발 중심 추정 | 실제 인원 확인 필요 |
| 2026 | 2명 | 대표 1, 엔지니어 1 | 제품 안정화와 실험 속도 확보 |

## 6. 숫자 입력이 아직 불확실한 항목 목록

아래는 현재 반드시 실제값 확인이 필요한 항목이다.

| 항목 | 현재 상태 | 왜 불확실한가 |
| --- | --- | --- |
| 예비창업자 여부 | 확인 필요 | repo/artifact에는 법인/사업자 등록 정보 없음 |
| 현재 매출 | 확인 필요 | 결제/정산 근거 없음 |
| 현재 영업이익/순이익 | 확인 필요 | 실제 지출 집계 자료 없음 |
| 자본금 | 확인 필요 | 법인 설립 여부 및 납입자본금 정보 없음 |
| 현재 팀 인원 | 확인 필요 | repo만으로 인원 구조 판단 불가 |
| 실제 API/클라우드 월 비용 | 확인 필요 | benchmark call count는 있으나 실제 청구서 없음 |
| 공간비/사무비 | 확인 필요 | 현재 작업 환경 비용 자료 없음 |
| 채용 급여 기준 | 확인 필요 | 채용 시점, 고용형태, 연봉 기준 미정 |
| 반복 사용률 current baseline | 확인값 없음 | interaction log 저장은 있으나 KPI 집계 체계 미완성 |
| 결제 의사/기관 파일럿 current baseline | 확인값 없음 | 아직 고객개발 결과 수치가 repo에 없음 |
